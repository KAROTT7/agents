# 布局与信息层级

用页面结构解决“看哪里、先做什么、如何继续”。

## 层级检查

- 首屏是否在 3 秒内说明页面用途。
- 主标题、核心指标、主 CTA 是否形成稳定焦点。
- 次级信息是否被压低，不与主任务争抢注意力。
- 相同层级的模块是否有一致的对齐、间距和标题逻辑。

## 常见页面骨架

### 仪表盘

- 顶部放页面标题、时间范围、关键筛选和主操作。
- 首屏放概览指标与异常提醒。
- 下方按任务流组织图表、明细表和辅助说明。

### 列表页

- 把筛选、搜索、批量操作、结果统计和表格关系排清楚。
- 让最常用操作距离数据最近，减少来回移动。

### 表单页

- 先分组，再决定单栏还是双栏。
- 让高风险动作、保存反馈和校验信息足够明确。

### 营销页

- 首屏聚焦价值主张、信任背书和核心 CTA。
- 用强弱节奏组织卖点，不要每一屏都同样吵。

## 响应式判断

- 移动端优先保留核心任务与核心信息，不机械缩小桌面布局。
- 桌面端利用横向空间建立并列关系，避免过长纵向滚动。
- 模块重排时先保留优先级，再决定形式。
