# UI 评审清单

按从高影响到低影响的顺序审查界面。

## 1. 任务清晰度

- 页面目标是否明确。
- 用户进入后第一步动作是否明显。
- 文案是否帮助理解，而不是制造歧义。

## 2. 视觉层级

- 标题、数据、操作、说明是否有清楚主次。
- 颜色、字重、尺寸是否在强化层级，而不是互相打架。
- 页面是否存在多个“假主角”。

## 3. 结构与节奏

- 区块划分是否清楚。
- 留白是否服务于阅读节奏，而不是单纯变空。
- 页面是否存在无意义重复组件或重复信息。

## 4. 组件一致性

- 相同按钮、输入框、卡片、标签是否风格统一。
- 图标、圆角、阴影、边框粗细是否系统一致。
- 状态反馈是否完整且可预测。

## 5. 可用性

- 关键操作是否足够接近使用场景。
- 错误提示、保存反馈、删除确认是否明确。
- 复杂任务是否被拆成更易理解的步骤。

## 6. 响应式与可访问性

- 移动端是否保留核心路径。
- 对比度、点击热区、键盘焦点是否可用。
- 颜色是否不是唯一状态区分方式。

## 7. 工程可落地性

- 样式是否能复用现有组件或 token。
- 是否引入了大量一次性特效或难维护结构。
- 视觉细节是否值得对应实现成本。

## 输出格式建议

- 严重问题：直接影响理解、操作或转化。
- 重要问题：明显降低质感、一致性或效率。
- 优化建议：可提升完成度，但不阻塞主流程。
