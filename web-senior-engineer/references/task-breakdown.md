# 任务拆解指南

将需求转化成工程可执行计划时，按“里程碑 + 任务 + 验收”组织。

## 推荐结构

- 里程碑名称
- 阶段目标
- 任务列表
- 前置依赖
- 风险点
- 验收标准

## 典型里程碑

1. 基础设施与项目初始化
2. 核心主流程开发
3. 次级功能补全
4. 质量加固与测试
5. 上线准备与交付

## 任务拆解规则

- 每个任务都应可观察、可验证
- 将调研类任务和实现类任务分开
- 显式标出阻塞项和依赖项
- 任务粒度应足以估时
- 容易产生歧义的任务必须带验收标准

## 常见分类

- 产品与设计
- 前端开发
- 后端开发
- 数据库与模型
- 认证与权限
- 测试与质量保障
- 分析埋点
- 运维与部署
- 发布与运营支持
