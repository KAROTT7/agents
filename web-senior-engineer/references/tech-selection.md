# 技术选型指南

技术选型的目标不是追求“最新”，而是在约束条件下找到最合适的方案。

## 评估维度

- 上线速度
- 团队熟悉度
- 维护成本
- SEO 要求
- 性能要求
- 实时能力需求
- 数据模型复杂度
- 安全与合规要求
- 预算与基础设施复杂度

## 推荐输出格式

对每个候选方案说明：

- 方案概述
- 优势
- 代价或限制
- 适用场景
- 为什么推荐或不推荐

## 常见默认建议

- 前端：如果需要快速迭代和完整 Web 能力，优先考虑 Next.js 或同类全栈框架
- 后端：普通业务系统优先使用 Node.js 生态或托管型后端
- 数据库：大多数事务型场景优先 PostgreSQL
- 鉴权：如果没有强定制需求，优先托管式认证方案
- 部署：根据团队规模和控制需求选择 Vercel、Cloudflare、AWS 等平台

## 选型原则

- 不要为了假设中的超大规模过早复杂化
- 小团队优先托管服务
- SEO 很重要时优先 SSR 或 SSG 能力强的框架
- 如果团队不熟悉某套基础设施，不要轻易引入
