# PRD 模板

适用于常见 Web 产品的需求文档结构。

## 1. 项目概述

- 产品名称
- 一句话简介
- 业务目标
- 计划上线时间

## 2. 问题定义

- 用户当前痛点
- 现有方案为什么不够好

## 3. 用户与场景

- 主要用户
- 次要用户
- 关键使用场景

## 4. 目标与指标

- 用户目标
- 业务目标
- 成功指标

## 5. 范围定义

- 本期范围
- 非本期范围
- 关键假设

## 6. 功能需求

每个功能建议包含：

- 用户故事
- 功能说明
- 关键流程
- 边界情况
- 验收标准

## 7. 非功能需求

- 性能
- 安全
- 可用性
- 可观测性
- 无障碍
- 国际化或多语言需求

## 8. 依赖与风险

- 技术依赖
- 业务依赖
- 主要风险
- 风险应对方式

## 9. 发布计划

- 里程碑
- 灰度或发布策略
- 上线判定标准
