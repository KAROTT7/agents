# web-senior-engineer 规格说明

## 目标

创建一个名为 `web-senior-engineer` 的 Codex skill，用于指导 Web 应用从产品定位到上线交付的完整流程。

## 适用人群

- 希望快速梳理产品方向的创业者
- 需要输出标准化 PRD 和交付计划的产品经理
- 需要清晰技术方案和实施路径的工程师

## 要解决的问题

- 需求常常在目标不清晰时就直接进入开发
- 产品范围、用户价值、优先级经常没有被明确写出来
- 技术选型容易停留在个人偏好，缺少权衡分析
- 任务拆解不够清晰，导致依赖遗漏、验收标准模糊
- 上线准备经常缺少完整的发布和回滚思考

## 范围

该 skill 应帮助 Codex：

- 分析产品定位与目标用户
- 明确业务目标、核心场景与成功指标
- 起草或完善 PRD
- 进行技术选型并说明取舍
- 将工作拆分为里程碑、任务、依赖与验收标准
- 输出上线准备与交付清单

## 非目标

- 代替安全、法务、合规等专业审核
- 强制规定唯一技术栈
- 产出适用于所有团队的重型流程体系

## 交付物

- `SKILL.md`
- `agents/openai.yaml`
- 产品定位、PRD、技术选型、任务拆解、上线清单等参考文档

## 成功标准

- 当用户提出做 Web 产品、梳理需求、输出 PRD、选型、排期、上线时能够触发
- 能按阶段执行，不强迫每次都走完整流程
- 输出内容偏实战、可执行、可交付
